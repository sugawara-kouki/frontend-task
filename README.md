## 概要

このリポジトリは、指定された技術課題の要件に基づき作成されたフロントエンドアプリケーションです。
バックエンド API と連携し、ページの閲覧、作成、編集、削除といった基本的な CRUD 機能を提供します。

## 技術スタック

- **フレームワーク**: Next.js (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **状態管理**: カスタムフック (useState, useEffect)
- **データフェッチ**: Fetch API
- **バリデーション**: Zod
- **テスト**: Jest + React Testing Library

技術選定の詳細な理由については、[基本設計 - アーキテクチャの決定事項](docs/design-basics.md#アーキテクチャの決定事項)を参照してください。

### 2. フロントエンドのセットアップ

このリポジトリ（フロントエンド）をセットアップします。

```bash
# 依存関係をインストール
npm install

# 開発サーバーを起動
npm run dev
```

アプリケーションは `http://localhost:3001` （または別の利用可能なポート）で起動します。ブラウザでアクセスしてください。

### アーキテクチャの概要

本アプリケーションは、保守性、可読性、拡張性を重視して設計しました。

**主な設計方針**:

- **関心の分離**: カスタムフック（`usePages`, `usePageEditor`）でビジネスロジックと UI を分離
- **責務の明確化**: プレゼンテーショナルコンポーネント、カスタムフック、共通ロジックで役割を分担
- **テスタビリティ**: 依存関係を props で注入し、ユニットテストが容易な構造

## テスト

本アプリケーションでは、以下のユニットテストを実装しています。

### 実装済みテスト

**バリデーションロジック** ([`app/lib/validation.test.ts`](app/lib/validation.test.ts)):

- Zod スキーマの境界値テスト（タイトル: 1〜50 文字、本文: 10〜2000 文字）
- エラーメッセージの検証

**カスタムフック** ([`app/hooks/usePageEditor.test.ts`](app/hooks/usePageEditor.test.ts)):

- エラー状態管理のテスト
- コールバック呼び出しの検証

### テストの実行

```bash
npm test
```

### テスト方針

UI から独立した純粋なロジック（バリデーション、状態管理）をユニットテストでカバーすることで、アプリケーションの堅牢性を担保しています。詳細は[基本設計 - テスト戦略](docs/design-basics.md#テスト戦略)を参照してください。
